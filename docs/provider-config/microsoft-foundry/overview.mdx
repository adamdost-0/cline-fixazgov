### Overview

**Microsoft Foundry** provides access to Azure AI Foundry (formerly Azure OpenAI Service), enabling Cline to use frontier AI models deployed in your Azure environment with enterprise-grade security, compliance, and governance.

**Supported Clouds:**
- **Azure Commercial** — Public Azure cloud
- **Azure Government** — US Government cloud (FedRAMP High, DoD IL4/IL5)
- **Azure Stack** — On-premises and edge deployments

**Commercial Website:** [Azure AI Foundry](https://ai.azure.com)
**Government Website:** [Azure Government AI](https://ai.azure.us)

---

### Why Use Microsoft Foundry?

| Feature | Benefit |
|---------|---------|
| **Enterprise Security** | Data stays in your Azure tenant with private endpoints, VNET integration, and customer-managed keys |
| **Compliance** | SOC 2, HIPAA, FedRAMP, and government cloud support |
| **Azure Identity** | Use your existing Azure CLI login — no separate API keys required |
| **Model Variety** | Access OpenAI models (GPT-5, GPT-4o), Claude, Llama, Phi, Mistral, and more |
| **Cost Control** | Provisioned throughput, token quotas, and Azure billing integration |

### Supported Models

Microsoft Foundry supports any model deployed to your Azure AI Foundry resource, including:

**OpenAI Models:**
- GPT-5.2, GPT-5.2-chat, GPT-5-mini, GPT-5-nano
- GPT-4o, GPT-4o-mini, GPT-4-turbo
- o3, o4-mini (reasoning models)

**Anthropic Claude (via Azure):**
- Claude Sonnet 4.5, Claude Opus 4, Claude Haiku 4

**Open Source Models:**
- Phi-4, Phi-4-mini, Phi-4-multimodal
- Llama 3.3 70B, Llama 4
- Mistral Medium, DeepSeek V3

<Note>
Model availability depends on your Azure region and deployment configuration. Check the [Azure AI Foundry Model Catalog](https://ai.azure.com/explore/models) for current availability.
</Note>

### Quick Start

#### Prerequisites

1. **Azure Subscription** with an Azure AI Foundry resource
2. **Deployed Model** in your Azure AI Foundry project
3. **Azure CLI** installed (for Azure Identity auth) or an API key

#### Get Your Endpoint URL

1. Go to [Azure AI Foundry](https://ai.azure.com)
2. Select your project
3. Navigate to **Deployments**
4. Copy the **Target URI** (e.g., `https://your-resource.cognitiveservices.azure.com`)

<Frame>
  <img src="/assets/azure-foundry-endpoint.png" alt="Finding your Azure AI Foundry endpoint" />
</Frame>

#### Configure Cline

1. Open Cline in VS Code
2. Click the settings gear ⚙️
3. Select **Microsoft Foundry** from the API Provider dropdown
4. Enter your **Endpoint URL**
5. Choose your **Cloud Environment** (Commercial, Government, or Stack)
   - For **Stack**: Also enter ARM Endpoint and Custom Scope (see Azure Stack Configuration below)
6. Select your authentication method
7. Click **Refresh Deployments** to find available models
   - **Azure Stack Note:** Automatic discovery is not available; enter deployment name manually
8. Select your deployment and start coding!

### Cloud Environments

<Tabs>
  <Tab title="Commercial">
    **Azure Commercial** is the standard public Azure cloud.
    
    - **Audience Scope:** `https://cognitiveservices.azure.com/.default`
    - **Endpoint Format:** `https://{resource-name}.cognitiveservices.azure.com`
    - **Regions:** All standard Azure regions
  </Tab>
  <Tab title="Government">
    **Azure Government** is the US Government cloud with enhanced compliance.
    
    - **Audience Scope:** `https://cognitiveservices.azure.us/.default`
    - **Endpoint Format:** `https://{resource-name}.cognitiveservices.azure.us`
    - **Compliance:** FedRAMP High, DoD IL4/IL5, CJIS, IRS 1075
    - **Regions:** US Gov Arizona, US Gov Virginia, US Gov Texas
    
    <Warning>
    Azure Government requires an approved government tenant. Contact your Azure administrator for access.
    </Warning>
  </Tab>
  <Tab title="Stack">
    **Azure Stack** enables on-premises and edge deployments with full control over infrastructure and data residency.
    
    - **Azure Resource Manager Endpoint:** Custom (e.g., `https://management.local.azurestack.external`)
    - **Audience Scope:** Custom (configured by your administrator)
    - **Endpoint Format:** Varies by deployment
    - **Use Case:** Air-gapped environments, data sovereignty requirements, edge computing
    
    <Warning>
    **Azure Stack Deployment Discovery Limitation**
    
    Azure Stack Hub does not support Azure Resource Graph API, which Cline uses for automatic deployment discovery. 
    You must **manually enter your deployment name** instead of using the "Refresh Deployments" button.
    </Warning>
    
    <Info>
    Azure Stack requires two custom values:
    1. **Azure Resource Manager Endpoint** — For infrastructure operations (e.g., `https://management.local.azurestack.external`)
    2. **Custom Audience Scope** — For Cognitive Services authentication (e.g., `https://cognitiveservices.local.azurestack.external/.default`)
    
    Contact your Azure Stack administrator for these values.
    </Info>
  </Tab>
</Tabs>
